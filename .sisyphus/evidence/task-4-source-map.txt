1. HedgePolicy is stored via the existing System key-value JSON pattern, not as dedicated ent schema fields. This follows the existing system settings architecture and is correct per the plan's "extend existing persistence path" requirement.

2. The system schema (system.go) does NOT contain hedge fields directly - it only stores key-value pairs. HedgePolicy is serialized to JSON and stored via the SystemService pattern. This is architecturally consistent with other policy storage (RetryPolicy, StoragePolicy, etc.).
